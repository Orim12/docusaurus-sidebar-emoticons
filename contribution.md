
# Contributing

Thank you for your interest in contributing!

- Fork the repository and create a feature branch from `master`.
- Make small, focused commits with clear messages.
- Add tests and/or examples for new functionality when possible.
- Follow the existing code style; format with Prettier/ESLint if present.
- Open an issue before making large changes, or link the PR to an existing issue.
- Describe in the PR what you changed and why, including reproducible steps.

By contributing you agree to the project's license.

If you have questions, open an issue or start a discussion in the repository.

**Repository files**

- [index.js](index.js): main script / entrypoint of the plugin or package.
- [package.json](package.json): project metadata, dependencies and scripts.
- [README.md](README.md): overview, installation and usage instructions.
- [setup.js](setup.js): setup or installation script (helper for initial configuration).
- [LICENSE](LICENSE): license terms for this project.
- [theme/Root.tsx](theme/Root.tsx): custom Docusaurus theme component (sidebar UI).
- [contribution.md](contribution.md): this contribution guide and guidelines.
